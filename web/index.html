Exemplo: Álbum da Copa

Você já tem 350 figurinhas únicas, 120 repetidas, pacote R$ 7,00

  • Faltam únicas = 980 − 350 = 630
  • Faltam efetivas = 630 − 120 repetidas = 510
  • H(510) = 6,87…
  • Figurinhas esperadas = 980 × 6,87 = 6.732,00
  • Pacotes esperados = ceil(6.732,00 / 7) = 962
  • Valor esperado = 962 × R$ 7,00 = R$ 6.734,00

Como o cálculo é feito?

Usamos o Problema do Colecionador de Cupons. O número de figurinhas esperadas é dado por:

1. Número Harmônico (exato)

H(n) = ∑ 1/i
     = 1 + 1/2 + 1/3 + … + 1/n
→

2. Figurinhas esperadas

Figurinhas = n × H(n)
↓

3. Pacotes esperados

Pacotes = ceil( Figurinhas / pacote )
(arredondado para cima)
→

4. Valor esperado

Valor = Pacotes × preço

❓ Como Funciona

Entenda o raciocínio matemático por trás da calculadora.

1

O Problema

Cada pacote contém figurinhas aleatórias. Quanto mais perto você estiver de completar, mais difícil fica encontrar as que faltam. Isso é o Problema do Colecionador de Cupons.

2

Número Harmônico

O valor esperado de compras para obter a k-ésima figurinha nova é n/(n−k+1). Somando todos os termos, chegamos ao Número Harmônico H(n).

3

Método Exato

Somamos 1 + 1/2 + 1/3 + … + 1/n para encontrar H(n) com precisão, sem usar a aproximação ln(n) + 0,5772… O resultado é multiplicado por n.

4

Custo Final

Com o total de figurinhas esperadas, calculamos os pacotes necessários (arredondados para cima, pois não existe meio pacote) e multiplicamos pelo preço.

🎯 Objetivo

Criar uma aplicação web capaz de calcular a quantidade esperada de figurinhas necessárias para completar um álbum, considerando distribuição uniforme, sem troca entre colecionadores, comprando apenas pacotes novos. O algoritmo é baseado no Problema do Colecionador de Cupons.
